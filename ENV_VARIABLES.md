# Environment Configuration Reference

## Frontend (Vercel) - Client

### Build Configuration
- **Framework**: Vite React
- **Build Command**: `npm run build`
- **Output Directory**: `dist`
- **Node Version**: 18 LTS or higher
- **Package Manager**: npm / bun

### Required Environment Variables

```env
VITE_API_URL
```

**Value for Production**: `https://your-railway-backend.railway.app`

### Optional Environment Variables

```env
VITE_DEBUG=false
```

---

## Backend (Railway) - Server

### Runtime Configuration
- **Language**: Python 3.11+
- **Start Command**: `python app.py`
- **Port**: `5000`
- **Process Type**: Web

### Required Environment Variables

```env
# Database
DB_HOST=<from Railway MySQL credentials>
DB_PORT=3306
DB_USER=<from Railway MySQL credentials>
DB_PASSWORD=<from Railway MySQL credentials>
DB_NAME=itsm

# Flask
FLASK_ENV=production
FLASK_DEBUG=False
PORT=5000

# API CORS
CORS_ORIGINS=https://your-vercel-frontend.vercel.app

# Security
SECRET_KEY=<generate random string>
```

### How to Generate Variables

#### Generate SECRET_KEY
```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

#### Get MySQL Credentials (Railway)
1. Go to Railway Dashboard
2. Select MySQL service
3. Click "Variables"
4. Copy: `MYSQLUSER`, `MYSQLPASSWORD`, `MYSQLHOST`

## Database (Railway MySQL)

### Initialization

When first setting up:

1. Connect to your Railway MySQL:
```bash
mysql -h {MYSQLHOST} -u {MYSQLUSER} -p{MYSQLPASSWORD}
```

2. Run initialization script:
```bash
mysql -h {MYSQLHOST} -u {MYSQLUSER} -p{MYSQLPASSWORD} < server/db_init.sql
```

3. Or import manually in SQL client using `server/db_init.sql`

### Connection Details
- **Host**: Railway provided (private network)
- **Port**: 3306
- **Database**: itsm
- **Credentials**: Auto-generated by Railway

## Setup Order for New Deployments

1. **Railway MySQL**
   - Create project
   - Note credentials
   - Set `DB_NAME=itsm`

2. **Railway Backend**
   - Connect GitHub repo
   - Set all environment variables
   - Deploy and test `/api/ping`

3. **Railway Database Init**
   - Initialize with `db_init.sql`
   - Verify tables created

4. **Vercel Frontend**
   - Connect GitHub repo
   - Set `VITE_API_URL`
   - Deploy and test connection

5. **Testing**
   - Test `/api/debug/connection`
   - Test frontend login
   - Monitor logs for errors

## Local Development vs Production

### Local Development
```
Frontend: http://localhost:8080 or http://localhost:5173
Backend: http://localhost:5000
Database: localhost:3306
FLASK_ENV: development
FLASK_DEBUG: True
CORS_ORIGINS: http://localhost:8080,http://localhost:5173
```

### Production
```
Frontend: https://your-project.vercel.app
Backend: https://your-backend.railway.app
Database: Railway MySQL endpoint
FLASK_ENV: production
FLASK_DEBUG: False
CORS_ORIGINS: https://your-project.vercel.app
```

## Troubleshooting Environment Issues

### API Connection Failed
- Check `VITE_API_URL` in Vercel environment
- Verify backend is running on Railway
- Test: `curl https://your-backend.railway.app/api/ping`

### Database Connection Error
- Verify all DB_* variables are set correctly
- Confirm credentials match Railway MySQL
- Check `DB_HOST` is not localhost in Railway

### CORS Error in Frontend
- Update `CORS_ORIGINS` to include your Vercel URL
- Restart backend after changing environment
- Ensure frontend URL matches exactly (including https://)
